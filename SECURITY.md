## Reporting a Vulnerability

We take security issues seriously. If you discover a vulnerability in Shiot, please follow these steps to report it:

1. **Do not publicly disclose the issue**: Avoid sharing the vulnerability in public forums, GitHub Issues, or pull requests.

2. **Contact us privately**:  
   Send an email to **[adrian.saca2004@gmail.com]** with the following details:
   - A detailed description of the vulnerability.
   - Steps to reproduce the issue.
   - Any potential impact the issue may have.
   - Suggested fixes (if available).

3. **Response Time**:  
   We aim to acknowledge receipt of the issue within **48 hours** and provide a resolution timeline, if applicable, within **7 days**.

---

## Security Best Practices for Contributors

When contributing to this repository, please adhere to the following guidelines to maintain a secure codebase:
- Validate all user inputs and sanitize external data.
- Avoid hardcoding sensitive information (e.g., API keys, credentials) in the codebase.
- Use HTTPS for all external API calls and communications.
- Report any third-party library vulnerabilities you encounter while working on the project.
